## 협업 규칙

### Git commit convention
- FIX: 잘못된 부분 수정
- CHORE: 빌드 업무 수정
- ADD: 기능 추가
- MOD: 코드 수정
- RM: 기능 삭제

### Toondot commit rule
- 자신의 레포지토리에 Origin 레포지토리를 fork한후 그곳에서 origin 레포지토리의 dev 브랜치에 커밋을 날린다.
- main merge는 각팀의 팀장이 한다.
